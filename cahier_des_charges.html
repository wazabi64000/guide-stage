Cahier des Charges

Le cahier des charges est un document fondamental qui définit les besoins du projet et les attentes du client. Il sert de guide pour toutes les étapes de votre stage.

1. Contenu essentiel

  • Objectifs du projet : que doit accomplir le projet ?
  • Fonctionnalités : liste détaillée de toutes les fonctionnalités attendues.
  • Contraintes : techniques, légales, délais et budget.
  • Livrables : ce qui doit être produit et validé par le client.
  • Planification : étapes et deadlines pour chaque fonctionnalité.

2. Exemples concrets

Exemple : Pour une application web de gestion de tâches :

  • Fonctionnalité 1 : Ajouter une tâche avec titre, description et priorité.
  • Fonctionnalité 2 : Modifier ou supprimer une tâche.
  • Fonctionnalité 3 : Filtrer les tâches par statut ou priorité.
  • Contraintes : Utiliser React pour le front-end et Firebase pour le back-end.
  • Livrables : Code source sur GitHub, documentation utilisateur, diagramme des fonctionnalités.

3. Astuces pour bien rédiger

  • Vulgarisez le langage technique pour que le client comprenne.
  • Validez chaque section avec le client ou tuteur avant de passer à l’étape suivante.
  • Incluez des schémas, diagrammes et maquettes pour clarifier les fonctionnalités.
  • Toujours documenter les modifications ou changements de périmètre.

4. Ressources et outils

  • Modèle de cahier des charges : téléchargeable en Word ou Google Docs. => Model
  • Outils pour diagrammes : Draw.io, Figma, Lucidchart.
  • Documentation : Notion, Google Docs, Word.
  • Versioning : Git/GitHub pour suivre les versions de votre cahier des charges.

Astuce : Le cahier des charges n’est pas figé : il doit évoluer avec le projet tout en restant validé par le client ou tuteur.
