Choix du Projet

Le projet que vous réalisez pendant le stage est crucial pour valider vos compétences et démontrer votre valeur à l’entreprise. Voici comment procéder selon différents scénarios.

1. L'entreprise ne propose pas de projet

  • Proposez un projet personnel pertinent, en lien avec le domaine de l’entreprise.
  • Assurez-vous que votre projet couvre toutes les compétences à valider.
  • Rédigez un petit document de présentation du projet et obtenez la validation du tuteur.

Exemple : Créer une application web de gestion de tâches avec React pour le front-end et Firebase pour le back-end.

Astuce : Utilisez un diagramme ou mindmap pour visualiser le projet et ses modules avant de le présenter.

Outils : Trello, Notion pour planification; Miro ou XMind pour le mindmapping.

2. L'entreprise propose un projet incomplet

  • Identifiez les compétences ou modules qui ne sont pas couverts par le projet proposé.
  • Proposez des extensions ou des fonctionnalités supplémentaires pour compléter votre apprentissage.
  • Documentez chaque étape pour montrer vos compétences techniques et organisationnelles.

Exemple : L’entreprise propose un site statique. Vous pouvez ajouter un module back-end pour gérer une base de données ou des utilisateurs.

Astuce : Préparez un plan en trois phases : fonctionnalités de base, extensions optionnelles, tests et validation.

Outils : GitHub Projects pour suivre vos tâches, Trello ou Jira pour organiser les étapes.

Conseils généraux pour choisir un projet

  • Choisissez un projet **réaliste mais ambitieux**, qui vous met au défi sans être impossible.
  • Assurez-vous que chaque fonctionnalité du projet peut être documentée et évaluée.
  • Préparez des livrables intermédiaires pour montrer vos progrès au tuteur.
